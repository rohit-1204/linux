# Why Study Linux History?

Understanding Linux history helps you:

- Understand why Linux works the way it does
- Appreciate open-source software
- Learn the philosophy behind Linux
- Understand modern operating systems

Many interview questions also ask about Linux's origins.

---

# Before Linux: The World of Computers

During the 1960s and early 1970s:

- Computers were extremely expensive.
- Each computer had its own operating system.
- Software written for one machine usually could not run on another.
- Operating systems were difficult to maintain.

Researchers wanted a simpler and more portable operating system.

---

# The Birth of UNIX (1969)

UNIX was created in **1969** at **Bell Labs**.

The main creators were:

- Ken Thompson
- Dennis Ritchie

Their goal was to build:

- Simple
- Efficient
- Multi-user
- Multitasking

operating system.

UNIX became one of the most influential operating systems ever created.

---

# Bell Labs

Bell Labs was one of the world's most famous research laboratories.

Many important technologies came from Bell Labs, including:

- UNIX
- C Programming Language
- C++
- Transistor technology
- Information theory

Bell Labs greatly influenced modern computing.

---

# Dennis Ritchie and the C Programming Language

Originally UNIX was written in Assembly language.

Later, Dennis Ritchie developed the **C Programming Language**.

UNIX was rewritten in C.

This made UNIX:

- Portable
- Faster to develop
- Easier to maintain

Today, Linux itself is mostly written in C.

---

# Why Was UNIX Revolutionary?

UNIX introduced many ideas that still exist today.

Examples:

- Everything is a file
- Small programs doing one job well
- Multi-user support
- Multitasking
- Pipes
- Command-line tools
- Hierarchical file system

These ideas remain central to Linux.

---

# UNIX Philosophy

The UNIX philosophy is simple:

> "Write programs that do one thing and do it well."

Instead of one huge application,

UNIX combines many small tools.

Example:

```bash
cat file.txt | grep "error" | sort
```

Each command performs a single task.

Together they solve a bigger problem.

---

# Commercial UNIX

UNIX became extremely popular.

Many companies created their own versions.

Examples:

- IBM AIX
- HP-UX
- Sun Solaris
- SCO UNIX

Problems appeared:

- Expensive licenses
- Different UNIX versions
- Limited availability

Developers wanted a free alternative.

---

# Richard Stallman and the GNU Project (1983)

In **1983**, Richard Stallman started the **GNU Project**.

Goal:

Create a completely free UNIX-like operating system.

GNU stands for:

> **GNU's Not UNIX**

The GNU Project created many important tools:

- GCC Compiler
- Bash Shell
- GNU Core Utilities
- GDB Debugger
- Make

However...

One important component was missing.

The kernel.

---

# Free Software Foundation (FSF)

Richard Stallman founded the **Free Software Foundation (FSF)** in 1985.

Its mission:

Protect software freedom.

The four freedoms are:

1. Run software
2. Study software
3. Modify software
4. Share software

---

# GNU General Public License (GPL)

The GPL is one of the world's most popular software licenses.

It allows anyone to:

- Use software
- Modify software
- Share software

But...

Any modified version must also remain open source.

This protects software freedom.

---

# MINIX

Before Linux existed,

Professor Andrew Tanenbaum created MINIX.

Purpose:

Teach operating system concepts.

MINIX looked similar to UNIX.

Limitations:

- Educational only
- Limited functionality
- Restricted licensing

Many students learned operating systems using MINIX.

One of those students was...

Linus Torvalds.

---

# Who is Linus Torvalds?

Linus Torvalds was a computer science student at the University of Helsinki, Finland.

He wanted:

- A better operating system
- Something more powerful than MINIX
- A free UNIX-like kernel

So he started writing one himself.

---

# Birth of Linux (1991)

In 1991,

Linus announced his project on the Internet.

His famous message began with:

> "I'm doing a free operating system..."

It was initially just a hobby project.

No one imagined it would change computing forever.

---

# Linux Version 0.01

Released:

September 1991

Characteristics:

- Around 10,000 lines of code
- Intel 80386 processors
- Very limited features
- Required technical knowledge

Today,

Linux contains tens of millions of lines of code.

---

# Why Was Linux Successful?

Several reasons contributed:

- Free
- Open source
- Community driven
- Reliable
- Portable
- Secure
- Continuous improvements

Thousands of developers worldwide contributed.

---

# GNU + Linux

The Linux kernel alone is not a complete operating system.

GNU already had:

- Bash
- GCC
- Core utilities
- Libraries

Combining:

GNU + Linux Kernel

created a complete operating system.

This is why some people call it:

GNU/Linux

---

# Open Source Development

Unlike proprietary software,

Linux development is open.

Anyone can:

- Read the code
- Report bugs
- Submit improvements
- Review changes

This creates rapid innovation.

---

# Linux Mascot - Tux

The Linux mascot is:

**Tux the Penguin**

Designed by:

Larry Ewing (1996)

Why a penguin?

Linus Torvalds liked penguins.

Tux became one of the most recognizable mascots in technology.

---

# Open Source vs Proprietary Software

| Open Source | Proprietary |
|-------------|-------------|
| Source code available | Source code hidden |
| Community contributions | Vendor controlled |
| Usually free | Usually paid |
| Customizable | Limited customization |
| Transparent | Closed development |
